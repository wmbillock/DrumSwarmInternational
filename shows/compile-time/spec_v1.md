# Compile Time

## Show Concept

**Core Theme:** A meditation on the tension between creation and execution, presented through the metaphor of software compilation.

**Three-Movement Arc:**

1. **Movement 1: "Source Code"** — Brass writes melodic fragments that build into a unified theme, like code being written line by line. Linear grid formations expand outward as each musical phrase adds a "line" of marchers. 8 marchers per line, expanding to 6 lines (48 total brass). Grid expands in a linear path (straight columns advancing outward).

2. **Movement 2: "Build Errors"** — Controlled chaos represents debugging. Deliberate dissonance, broken formations that reassemble, jagged diagonals fragmenting then snapping into clean perpendicular patterns. Guard flags execute horizontal figure-8s during dissonance section, with silk mirroring jagged drill angles. Green catches sync reassembly moments. Guard maintains red visual state transitioning to green mid-recovery.

3. **Movement 3: "Deployment"** — Triumphant full-ensemble resolution. Concentric circles converge to center point, guard integrates fully with formations as circuit data flowing through the completed circuit. The theme crystallizes in unison.

**Visual Metaphor:** Drill formations evoke circuit board patterns and data flow diagrams. Guard flags act as visual markers of system state (error/success). Overall design language unifies technology imagery across movement, drill, music, and visual design.

---

## Musical Design

**Compositional Architecture:**

- **Movement 1:** Open voicings built on fourths and fifths that stack like code compilation. Piccolo/clarinet unison enters at 120 BPM as foundational voice. Trombone fourths enter underneath at bar 8, layering progressively. Brass tutti crystallizes at 1:45, with full key shift up a fourth at Movement 2 entry. Clean harmonic contrast establishes foundation for chaos ahead.

- **Movement 2:** Deliberate dissonance and fragmented phrasing mirror debugging process. Suggests controlled use of cluster chords, rhythmic disruption, and selective silence for maximum tension. Guard flag transitions and reassembly moments provide visual/sonic sync points.

- **Movement 3:** Full ensemble statement resolves all harmonic tension from Movement 2's key shift. Unison/octave doublings create triumphant clarity and completion.

**Key Principle:** Voicing stacking creates literal musical representation of compilation process—individual lines building toward executable unity.

**Duration Targets:**
- Movement 1: 3 minutes
- Movement 2: 4 minutes
- Movement 3: 3 minutes

## Visual Design

**Drill Formation Arc:**

- **Movement 1:** Linear grid formations expanding outward in straight columns. 8×1 marcher configuration expands to 8×6 (48 total brass). Each brass musical phrase triggers a new "line" of marchers entering the grid. Density increases progressively, visually representing code accumulation.

- **Movement 2:** Jagged diagonal patterns fragment the grid. Formations break symmetry deliberately, then reassemble into clean perpendicular patterns—visual metaphor for error correction and optimization. Guard flag figure-8s echo jagged angles during dissonance section.

- **Movement 3:** Concentric circles converge to a single center point. Represents compilation completion and deployment success. Guard integrates fully, weaving between formations as data flowing through the circuit.

**Circuit Imagery:** All formations should subtly reference electrical circuit patterns, printed circuit board topography, and data flow diagrams where formations intersect or traverse.

**Precision Timing:** Formation transition tolerance = 4 counts (approximately 2 seconds at 120 BPM).

---

## Guard Design

**Flag State Progression:**

- **Movement 1 → Movement 2 Transition:** Flags shift from neutral to red (error state). Guard maintains red visual state while executing horizontal figure-8s during dissonance section to mirror jagged drill angles and trace error cascades.

- **Movement 2:** Silk catches red light during fragmented drill. Guard formations mirror error states, positioned asymmetrically within drill patterns. Figure-8 patterns emphasize sudden directional changes. Guard flag arcs align with formation reassembly moments, providing visual feedback of recovery progression.

- **Movement 2 → Movement 3 Transition:** Green light catches mid-recovery. Flags transition to green, representing successful debugging and optimization.

- **Movement 3:** Guard maintains green visual, integrating fully with formations as circuit data. Guard weaves between converging concentric circles like traversing a completed circuit, flowing through the final formations. Final position creates visual closure with drill convergence.

**Technical Execution:** 
- Flag arcs emphasize sudden directional changes and recoveries
- Horizontal figure-8s execute during Movement 2 dissonance section
- Silk material allows clean color distinction and catch potential across all three movements
- Guard integrates into drill formations (not visually separate) in Movement 3
